Build "muv" — a natural language CLI for end users to interact with the Movement blockchain (Aptos fork, chain ID 126) via plain English.

PRODUCT: Users type natural language like "swap 10 USDC.e for MOVE on Meridian" or "what's my balance?" and muv translates intent into on-chain actions using Claude API.

TECH STACK: TypeScript, npm package named "muv", command "muv"
- Claude API via @anthropic-ai/sdk for NL understanding
- Aptos TS SDK (@aptos-labs/ts-sdk) for chain interaction
- Movement mainnet RPC: https://mainnet.movementnetwork.xyz/v1

CORE FEATURES (v1):
1. WALLET MANAGEMENT: On first run, prompt user to import private key/mnemonic OR generate a new wallet. Store plaintext in ~/.config/muv/wallet.json

2. PERSONALITY SELECTION: On first run, let user choose communication style: (1) Keep it short / terse (2) Be conversational / friendly. Store preference in config.

3. NATURAL LANGUAGE INTERFACE: Interactive REPL mode. User types plain English, Claude API parses intent and maps to blockchain actions. On ambiguous input, AI asks clarifying questions.

4. MERIDIAN DEX INTEGRATION (v1 scope):
   - Swaps via AMM (0xfbdb3da73efcfa742d542f152d65fc6da7b55dee864cd66475213e4be18c9d54)
   - Swaps via CLAMM (0x88def51006db6ae8f90051a1531d1b43877eeb233f4c0d99dcb24f49cd27ad5b)
   - Farming/LP positions (0xf1fc2bc72b9eeaa3cc80239d5c00e49ebab0b2c8a5b55227ce47644b3275ff96)
   - CLAMM Farming (0x4c5da52eaa510af14e93e7b16dddf3c5d6a9b3f847d18dc8e7499fc71a5a0a24)

5. NATIVE MOVE TRANSFERS: Send MOVE or any supported token to an address.

6. READ QUERIES (deep): Balance checks (all tokens), Meridian pool positions, token prices, transaction history.

7. MANDATORY TX CONFIRMATION: Before any transaction, show the full payload (action, amounts, gas estimate, recipient) and require y/n confirmation.

8. HARDCODED TOKEN REGISTRY: Use these EXACT token addresses and decimals in src/data/tokens.ts. Do NOT fabricate addresses.

IMPORTANT CONTEXT:
- Movement is an Aptos fork. The Aptos TS SDK (@aptos-labs/ts-sdk) works with Movement when configured with the Movement RPC endpoint.
- Meridian is a CLMM (Concentrated Liquidity Market Maker) DEX on Movement.
- The token registry should include all tokens from the provided list with their faAddress, symbol, name, and decimals.
- Native MOVE uses the standard Aptos native coin address (0x1::aptos_coin::AptosCoin or fungible asset equivalent).
- All transaction operations MUST show a confirmation prompt before submission.
- The AI should use Claude tool_use / function calling to map NL input to structured actions.
